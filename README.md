# Electronic Pass Management System

Учебное веб-приложение для управления пользователями, электронными пропусками и журналом событий доступа.

Проект разработан на Node.js, Express, MongoDB и EJS. Приложение позволяет создавать пользователей и пропуска, регистрировать входы и выходы, просматривать историю событий и формировать отчёты.

## Основные возможности

- создание пользователей;
- редактирование данных пользователей;
- удаление пользователей;
- создание электронных пропусков;
- привязка пропуска к определённому пользователю;
- редактирование и удаление пропусков;
- проверка корректности периода действия пропуска;
- регистрация входа и выхода по пропуску;
- запрет регистрации события по просроченному пропуску;
- удаление связанных записей журнала при удалении пропуска;
- фильтрация журнала событий по пользователю;
- фильтрация журнала по типу действия;
- фильтрация журнала по периоду;
- просмотр сводной информации о пользователях, пропусках и событиях;
- формирование и скачивание отчёта в формате TXT;
- серверный рендеринг страниц с помощью EJS;
- хранение данных в MongoDB.

## Модели данных

### User

Содержит информацию о пользователе:

- имя;
- адрес электронной почты.

### Pass

Содержит информацию об электронном пропуске:

- владелец пропуска;
- тип пропуска;
- дата начала действия;
- дата окончания действия.

### Log

Содержит информацию о событии доступа:

- пользователь;
- использованный пропуск;
- действие: вход или выход;
- дата и время события.

## Пользовательские сценарии

### Управление пользователями

Пользователь системы может:

1. Добавить нового пользователя.
2. Изменить его имя или электронную почту.
3. Удалить пользователя.

### Управление пропусками

Пользователь системы может:

1. Выбрать владельца пропуска.
2. Указать тип пропуска.
3. Задать даты начала и окончания действия.
4. Отредактировать данные пропуска.
5. Удалить пропуск.

Перед сохранением приложение проверяет корректность указанных дат и не разрешает установить дату окончания раньше даты начала.

### Журнал событий

Пользователь системы может:

1. Выбрать действующий пропуск.
2. Зарегистрировать вход или выход.
3. Просмотреть историю событий.
4. Отфильтровать записи по пользователю.
5. Отфильтровать записи по типу действия.
6. Ограничить выборку определённым периодом.

### Отчёты

Приложение объединяет информацию о пользователях, их пропусках и событиях доступа.

Сформированный отчёт можно просмотреть в браузере или скачать в формате TXT.

## Локальный запуск

### Требования

Для запуска проекта потребуются:

- Node.js;
- npm;
- MongoDB.

### 1. Клонирование репозитория

```bash
git clone https://github.com/KARAMELKA07/project-directory.git
cd project-directory
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка MongoDB

Убедитесь, что MongoDB установлена и запущена.

В файле `server.js` найдите переменную:

```javascript
const MONGO_URI = 'mongodb://192.168.198.219:27017/passSystem';
```

Замените указанный IP-адрес на адрес своей базы данных.

Для локально запущенной MongoDB можно использовать:

```javascript
const MONGO_URI = 'mongodb://127.0.0.1:27017/passSystem';
```

База данных `passSystem` будет использоваться для хранения пользователей, пропусков и записей журнала.

### 4. Запуск приложения

```bash
node server.js
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:3000
```
